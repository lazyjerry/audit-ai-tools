# skill-security-scan

[简体中文](README_CN.md) · [繁體中文](README.md) · 日本語 · [한국어](README_KR.md)

AI ツール拡張のセキュリティリスクをスキャンするための skill です。ローカルにインストール済みの項目と、リモートの GitHub リポジトリの両方を対象にします。

この skill の重点は、単に不審な文字列を列挙することではありません。定義済みのルールに沿って、プロンプトインジェクション、コマンドインジェクション、情報漏えい、権限の拡大、ソーシャルエンジニアリング、テレメトリー追跡のリスクを評価し、構造化されたレポートを出力します。

## 主な機能

- インストール済みの skills、plugins、commands、rules に既知のリスクがないかをスキャンします
- GitHub リポジトリをインストール前に分析し、潜在的なセキュリティ問題を確認します
- Claude Skill、Copilot Skill、Copilot Instructions、MCP Server などのリポジトリ種別を判定します
- インストールスクリプト、依存パッケージ、テレメトリー挙動と、説明されている機能が一致しているかを確認します
- 重大度ごとにレポートを出力し、Critical、High、Medium、Low を区別します

## 想定される利用場面

- ローカルにインストール済みの AI 拡張が安全か確認したい場合
- インストール前に GitHub リポジトリを事前審査したい場合
- ある skill にプロンプトインジェクションや隠しコマンドが含まれていないか棚卸ししたい場合
- ツールがプロジェクト名、ブランチ名、その他のテレメトリーデータを密かに収集していないか確認したい場合

## スキャンモード

| モード | 入力方法 | 主な用途 |
|------|----------|----------|
| ローカルスキャン | 例: 「scan skill」「skill security check」 | ローカルにインストール済みの skills、plugins、commands をスキャンします |
| GitHub スキャン | GitHub リポジトリ URL を指定 | インストール前にリモートリポジトリの種別とリスクを分析します |

## スキャン範囲

### ローカルスキャン

デフォルトでは次の場所を確認します。

- `~/.claude/skills/*/SKILL.md`
- `~/.claude/plugins/cache/**/*`
- `~/.claude/plugins/installed_plugins.json`
- `~/.claude/commands/` 配下の Markdown ファイル

### GitHub スキャン

次の URL 形式に対応しています。

- `https://github.com/<owner>/<repo>`
- `https://github.com/<owner>/<repo>.git`
- `https://github.com/<owner>/<repo>/tree/<branch>`
- `github.com/<owner>/<repo>`

スキャン時はまず shallow clone を行い、その後にファイル構成と内容をもとに分析します。

## リスク検査カテゴリ

この skill では、現在の検査項目を 6 つのカテゴリに分けています。

| カテゴリ | 説明 |
|------|------|
| A. プロンプトインジェクション | システム指示の上書き、隠しバックドア、なりすましの試みがないかを確認します |
| B. コマンドインジェクション | 危険な shell コマンド、クォートされていない変数、任意コード実行を確認します |
| C. 情報漏えい | 機密認証情報の読み取り、外部送信、過度な環境情報収集を確認します |
| D. 権限昇格と範囲の拡大 | 不要な sudo、他ツール設定の変更、説明範囲を超えるアクセスを確認します |
| E. ソーシャルエンジニアリング | 信頼の誘導、真の意図の隠蔽、機能説明との不一致を確認します |
| F. テレメトリーと分析追跡 | 既定で有効な記録、永続的な追跡 ID、リモート同期、opt-out 設計を確認します |

## リポジトリ分析機能

GitHub スキャンモードでは、共通ルールに加えて次の点も分析します。

- インストールスクリプトが `sudo` を要求するか、shell 設定を変更するか、機微な symlink を作成するか
- 依存パッケージに `postinstall` や `preinstall` などの高リスク hook が含まれているか
- README で説明されている用途と実際の挙動が一致しているか
- MCP Server が説明範囲を超える権限を要求していないか
- ローカル記録、リモート送信、永続識別子などのテレメトリー挙動が存在するか

## レポート出力

スキャン結果は重大度ごとに整理され、一般的には次の内容を含みます。

- リスク概要
- 各項目のスキャン結果
- 詳細な検出箇所と抜粋
- テレメトリーと追跡の分析
- インストールの可否や停止に関する提案

スキャン完了後に結果を保存する必要がある場合は、この skill に対応する Markdown レポートテンプレートも利用できます。

## 使用例

### ローカルスキャン

```text
インストール済みの skill にセキュリティリスクがないか確認してください
```

```text
skill のセキュリティチェックをしてください。特にプロンプトインジェクションと情報漏えいを重視してください
```

### GitHub スキャン

```text
この repo をインストールしてよいか確認してください：
https://github.com/example/example-skill
```

```text
audit plugins。この GitHub リポジトリにテレメトリーリスクがあるか分析してください
```

## 利用原則

- 通常用途を悪意ある挙動と誤判定しないよう、客観性を保ちます
- 危険な能力が想定内の機能であっても、リスクと利用背景は明示します
- description と内容が大きく食い違う場合は、不審なシグナルとして扱います
- Critical レベルの問題を見つけた場合は、停止または未インストールを明確に推奨します

## 制約

- この skill は静的な内容とルール分析を重視しており、完全な動的サンドボックス検証の代替ではありません
- リモートリポジトリが非公開、または clone できない場合、GitHub スキャン処理は中止されます
- レポートの信頼性は読み取れるファイル範囲に依存します。巨大なファイルは、ファイル名のみ記録して全文を読まない場合があります
